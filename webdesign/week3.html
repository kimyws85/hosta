화면에서 시작하는 순간

디자인이 화면에서 시작된다고 믿는 순간, 사고는 이미 가장 뒤쪽에서 출발하게 된다.

화면은 언제나 결과이며, 결과는 원인을 설명하지 않는다. 이 주차가 다루는 핵심은 디자인의 출발점을 시각적 구성에서 끌어내려, 문제 정의라는 보이지 않는 지점으로 되돌리는 것이다.

무엇을 어떻게 그릴 것인가보다 먼저, 무엇이 잘못 정의되었는지를 인식하지 못하면 그 이후의 모든 디자인 판단은 방향을 잃는다.

이 주차가 강제로 묻는 질문

01 지금 고치려는 것은 화면인가, 문제 정의인가
02 사용자는 어떤 상태로 진입했고, 어떤 판단이 막혔는가
03 이 화면은 어떤 목표와 제약 위에서 만들어졌는가

이 주차는 기술을 배우는 시간이 아니라, 디자인 사고의 출발점을 문제 정의로 고정하는 시간이다.

실패를 포장하는 착시

오해

“실행력/기술이 부족해서 실패한다”

디자인 실패를 생산성 문제로 환원
화면을 더 다듬으면 해결된다고 착각
구조의 오류를 시각 품질로 덮음

원인이 흐려진다

핵심

“문제 정의가 잘못되어서 실패한다”

잘못된 문제를 성실하게 해결하려 함
정렬/색/인터랙션이 실패를 그럴듯하게 포장
화면 개선이 구조 개선을 대체하지 못함

착시 제거

핵심 통찰

잘 정렬된 레이아웃, 세련된 색상, 부드러운 인터랙션은 문제를 해결하지 못한 상태에서도 “뭔가 해낸 것처럼 보이게” 만든다.

이 착시는 디자인을 화면 문제로 오해할 때 가장 강하게 발생한다.

화면 문제와 디자인 문제의 구분

화면 문제

“정리가 안 되어 있다”, “복잡하다”, “눈에 잘 안 들어온다”

디자인 문제

사용자의 진입 상태, 목적, 판단 정체 지점을 묻는 구조적 질문

화면 문제는 결과의 표현이고, 디자인 문제는 그 이전 단계에서 발생한다. 사용자가 어떤 상태로 진입했고, 무엇을 해결하려 했으며, 그 과정에서 어떤 판단이 막혔는지를 규정하지 못하면 화면 수정은 방향을 잃는다.

‘화면 수정’이 대체할 수 없는 것

문제 정의 무엇이 막혔는지의 정확한 규정
판단 경로 사용자가 어떤 순서로 결정하는지
우선순위 무엇을 먼저 해결해야 하는지의 선택

화면은 구조가 드러난 형태일 뿐이며,

화면을 고치는 일은 구조를 고치는 일의 대체물이 될 수 없다.

디자인을 움직이는 세 가지 힘

디자인은 단일한 요구로 움직이지 않는다. 디자인은 언제나 최소 세 가지 힘의 교차점에서 시작된다. 사용자 요구, 비즈니스 목적, 제약 조건이다.

사용자 요구
비즈니스 목적

문제 정의를 날카롭게 만드는 범위 규정

요구 하나로 화면을 만든다

3가지 힘의 교차점에서 문제 정의를 고정한다

잘못된 문제 정의의 치명성

잘못된 문제 정의 위에서의 개선은 문제를 더 빠르고 매끄럽게 유지시킬 뿐이다.

문제를 잘못 규정하면, 가장 뛰어난 실행력과 세련된 화면을 동원해도 결과는 어긋난다. 더 나은 버튼, 더 깔끔한 레이아웃, 더 부드러운 인터랙션은 잘못된 문제를 더 그럴듯하게 유지시킬 뿐이다.

치명성이 발생하는 이유

이후의 모든 선택이 문제 정의를 전제로 이루어짐
시각 품질이 실패를 “성공처럼” 보이게 만들 수 있음
화면 개선이 구조 오류를 숨기면 검증이 지연됨

전환 질문

지금 해결하려는 것은 증상인가, 목적 상태인가?
유도되는 행동이 서비스 목표와 일치하는가, 충돌하는가?
제약 안에서 무엇을 우선 해결해야 하는가?

과제 안내

과제는 착시를 제거하기 위해 설계되어 있다. 기존 웹 서비스를 하나 선택하는 이유는, 일정 수준의 완성도를 갖춘 화면 뒤에 어떤 문제 정의가 숨어 있는지를 추적하기 위함이다.

중요한 것은 화면을 비판하는 것이 아니라, 그 화면이 어떤 문제 정의 위에 세워졌는지를 해체하는 것이다.

빠른 구매 / 정보 비교 / 리스크 회피 / 확신 확보 등 ‘상태’로 고정

이 주차의 의미

이 주차는 디자인을 시작하는 법을 배우는 시간이 아니다. 어디서 시작하면 안 되는지를 몸으로 인식하는 시간이다.

화면에서 시작하는 사고를 멈추고 문제 정의로 되돌아갈 수 있을 때, 이후의 모든 디자인 선택은 이유를 갖는다.

화면에서 시작하면

잘못된 문제를 더 그럴듯하게 유지할 수 있음

문제 정의에서 시작하면

디자인은 “잘 만들기”가 아니라 “무엇을 푸는가”로 이동

지금의 선택은 어떤 문제를 해결하기 위해 존재하는가?

이 질문에 답할 수 있을 때에만, 화면은 의미 있는 결과로 작동한다.
