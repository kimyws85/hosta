C언어의 기본

프로그래밍 언어 : 사람이 생각한 절차(알고리즘)를 규칙(문법)으로 표현해, 컴퓨터가 실행 가능한 형태로 전달하는 형식 언어

소스 코드 : 사람이 작성하는 원본 코드 파일 (.c). 컴퓨터가 직접 실행하지 못하고 번역이 필요하다.

컴파일 : 소스 코드를 기계어로 변환 가능한 형태로 처리하는 과정(전처리 → 컴파일 → 어셈블 → 링크)

컴파일러 : C 소스 코드를 컴퓨터가 이해하는 기계어(또는 오브젝트 코드)로 번역하는 도구

링커(Linker) : 여러 오브젝트 파일과 라이브러리를 결합하여 하나의 실행 파일을 만든다

라이브러리 : 자주 쓰는 기능을 미리 만들어 둔 코드 묶음(표준 라이브러리, 외부 라이브러리)

실행 파일 : 운영체제가 직접 실행하는 결과물(.exe 등). 실행 시 main 함수부터 시작한다.

C언어의 특징

(1) 절차지향 언어 — “무엇을 할지”보다 “어떤 순서로 할지”를 코드로 고정한다.

(2) 이식성 — 같은 C 코드라도 컴파일 환경만 맞추면 다양한 OS/CPU에서 동작 가능하다.

(3) 저급 제어 — 메모리 주소/포인터/비트 연산 등 하드웨어에 가까운 제어가 가능하다.

(4) 성능과 예측 가능성 — 추상화가 얇아 오버헤드가 적고, 동작을 “내가 통제”하기 쉽다.

(5) 표준의 힘 — ANSI C / ISO C 표준을 기반으로 문법과 표준 라이브러리가 정리되어 있다.

(6) 단점(책임이 큼) — 메모리 관리, 경계 체크 등을 개발자가 직접 책임져야 한다.

C언어는 어디에 쓰이나

• 운영체제/드라이버/임베디드/펌웨어 등 “자원 제약이 있고, 성능·제어가 중요한” 영역

• 게임 엔진, 고성능 라이브러리, 네트워크/DB 내부 구성 요소 등 기반 기술

• 다른 언어의 런타임/확장 모듈(Python C 확장 등) 구현

컴파일(빌드) 흐름

1) 전처리(Preprocess) : #include, #define 같은 지시문 처리, 헤더 파일 내용 포함

2) 컴파일(Compile) : C 문법을 검사하고, 중간 표현/어셈블리로 변환

3) 어셈블(Assemble) : 어셈블리 코드를 오브젝트 파일(.obj/.o)로 변환

4) 링크(Link) : 오브젝트 + 라이브러리 결합 → 실행 파일 생성

개발 환경에서 자주 하는 기본 세팅(실무적)

• 경고 수준을 올려서(Warning Level) “위험 신호”를 조기에 잡는다.

• 입력 함수 사용 시 버퍼 오버플로우를 방지하는 방식으로 작성한다(길이 제한).

• 컴파일 옵션(/W4 등)과 표준(C11/C17)을 프로젝트 단위로 고정한다.

기초 구조 / 주석

기본 구조(가장 최소 형태) 
#include <stdio.h>     // 표준 입출력 함수 선언(printf, scanf 등)

  int main(void) {         // 프로그램 시작 지점(엔트리 포인트)
      printf("hello");      // 화면에 문자열 출력
      return 0;             // 운영체제에 "정상 종료"를 알림
  }

주석은 사람이 읽기 위한 설명이며, 컴파일 결과물에 포함되지 않는다. 코드가 길어질수록 “왜 이렇게 했는지”를 남기는 역할이 중요해진다.

// 한 줄 주석 : 해당 줄 끝까지 적용

/* 여러 줄 주석 */ : 범위를 지정해 여러 줄에 적용

main 함수 / return 값의 의미

운영체제는 프로그램을 실행하면 main 함수부터 호출한다. main의 반환값(return)은 운영체제에 “종료 상태 코드”를 전달한다.

일반적으로 return 0은 정상 종료, 0이 아닌 값은 오류/비정상 종료를 의미하도록 약속해서 사용한다.

헤더 파일(#include)의 역할

C는 함수/구조체 등을 사용하기 전에 “선언”이 필요하다. 표준 라이브러리 함수(printf 등)를 쓰려면 해당 선언이 들어 있는 헤더를 포함해야 한다.

자주 쓰는 헤더 예시 
#include <stdio.h>   // printf, scanf
  #include <stdlib.h>  // malloc, free, exit
  #include <string.h>  // strlen, strcpy, strcmp

출력 함수(printf)와 서식 지정자

printf는 단순 출력이 아니라, 형식(format)을 지정해 값을 문자열로 변환해 출력한다. 자료형과 서식 지정자가 맞지 않으면 출력이 깨지거나 예측 불가 동작이 생길 수 있다.

printf 예제 
#include <stdio.h>

  int main(void) {
      int n = 10;
      double pi = 3.14159;
      char ch = 'A';
      char s[] = "C언어";

      printf("정수: %d\n", n);
      printf("실수: %.2f\n", pi);
      printf("문자: %c\n", ch);
      printf("문자열: %s\n", s);

      return 0;
  }

자주 쓰는 서식 지정자

• %d : int 정수

• %f : float/double 실수(printf에서 float는 double로 승격되어 처리되는 경우가 많다)

• %c : 문자(char)

• %s : 문자열(char 배열, char*)

• %p : 포인터 주소(주소 출력)

• \n : 줄바꿈(개행). 출력 버퍼 플러시 타이밍에 영향이 생길 수 있다.

입력 함수(scanf) 기본 + 안전하게 쓰는 습관

입력은 출력보다 위험하다. 이유는 “사용자가 어떤 길이/형식으로 넣을지”를 코드가 통제하기 어렵기 때문이다. 따라서 길이 제한을 걸고, 반환값(성공 여부)을 확인하는 습관이 필요하다.

scanf 기본 예제(반환값 체크) 
#include <stdio.h>

  int main(void) {
      int x;
      printf("정수 입력: ");

      if (scanf("%d", &x) != 1) {
          printf("입력 형식이 올바르지 않습니다.\n");
          return 1;
      }

      printf("입력한 값: %d\n", x);
      return 0;
  }
문자열 입력(길이 제한) 
#include <stdio.h>

  int main(void) {
      char name[20];

      printf("이름 입력(최대 19자): ");
      // %19s : 공백 전까지 최대 19글자만 읽고 마지막에 \0을 붙인다
      if (scanf("%19s", name) != 1) {
          printf("입력 실패\n");
          return 1;
      }

      printf("안녕하세요, %s\n", name);
      return 0;
  }

중요 : 문자열 입력은 길이 제한이 없으면 위험하다. 버퍼(배열)보다 길게 들어오면 메모리 침범(버퍼 오버플로우)이 발생한다.

scanf 반환값 : 성공적으로 읽은 항목 개수를 반환한다. 입력 검증의 최소 장치다.

변수와 자료형(핵심만 정확히)

C에서 변수는 “값”이 아니라 메모리 공간이다. 자료형은 그 공간을 “어떤 크기/규칙으로 해석할지”를 결정한다. 자료형이 틀리면 같은 비트라도 의미가 달라진다.

기본 자료형 예시 
int a = 10;        // 정수
  double b = 3.14;   // 실수
  char c = 'Z';    // 문자(정확히는 1바이트 정수)

메모리와 주소(포인터 맛보기)

C의 강점은 “값”뿐 아니라 그 값이 저장된 주소를 직접 다룰 수 있다는 점이다. 주소를 다루면 배열/문자열/동적 할당/함수 인자 전달 방식이 모두 연결된다.

주소 출력 예제 
#include <stdio.h>

  int main(void) {
      int x = 10;
      int* p = &x;  // p는 x의 주소를 가진다

      printf("x 값: %d\n", x);
      printf("x 주소: %p\n", (void*)&x);
      printf("p 값(=주소): %p\n", (void*)p);
      printf("*p 값: %d\n", *p);

      return 0;
  }

& : 변수의 주소(address-of) 연산자

* : 포인터가 가리키는 값을 가져오는(역참조) 연산자

%p : 주소 출력. printf에서는 보통 (void*)로 캐스팅해서 출력하는 습관을 쓴다.

문자열과 배열(초기 감각)

C에서 문자열은 별도 타입이 아니라, char 배열 + 종료 문자 '\0'의 규칙이다. 따라서 문자열 관련 오류는 대부분 “끝 표시(\0) 누락” 또는 “배열 범위 초과”에서 나온다.

문자열의 실체 
#include <stdio.h>

  int main(void) {
      char s1[] = "ABC";     // {'A','B','C','\0'}
      char s2[4] = {'A', 'B', 'C', '\0'};

      printf("%s\n", s1);
      printf("%s\n", s2);
      return 0;
  }

초기에 가장 많이 터지는 실수

• 세미콜론( ; ) 누락

• 함수/변수 선언 위치 혼동, 괄호/중괄호 짝 불일치

• printf 서식 지정자와 자료형 불일치

• scanf에서 & 누락(정수/실수 입력 시)

• 문자열 입력 시 길이 제한 없이 %s 사용

• 배열 범위 초과 접근(인덱스 실수)

• 초기화하지 않은 변수 사용(쓰레기 값)

빠른 실습(기본 감각 고정)

1) 이름(문자열)과 나이(정수)를 입력받아 한 줄로 출력한다.

2) 정수 2개를 입력받아 합/차/곱을 출력한다.

3) 하나의 정수를 입력받아 짝/홀을 판단해 출력한다.

실습 예시(입력 + 출력 + 분기) 
#include <stdio.h>

  int main(void) {
      char name[20];
      int age;

      printf("이름(최대 19자): ");
      if (scanf("%19s", name) != 1) return 1;

      printf("나이: ");
      if (scanf("%d", &age) != 1) return 1;

      printf("입력 결과: %s (%d세)\n", name, age);

      if (age % 2 == 0) printf("나이는 짝수입니다.\n");
      else printf("나이는 홀수입니다.\n");

      return 0;
  }

핵심 정리

• C는 “흐름을 순서로 고정”하는 언어이며, 메모리/주소를 직접 다룰 수 있다.

• 빌드는 전처리→컴파일→어셈블→링크 흐름으로 실행 파일을 만든다.

• 출력은 서식 지정자, 입력은 길이 제한과 반환값 체크가 안전의 시작이다.

• 문자열은 char 배열 + '\0' 규칙이며, 배열 범위 초과가 가장 흔한 사고다.

변수 / 자료형 / 입출력 변수 선언, 자료형, 상수, printf, scanf
변수 / 자료형 / 상수

변수 : 메모리 공간에 붙여진 이름으로, 프로그램 실행 중 값을 저장·변경하기 위한 수단이다.

C에서 변수는 단순한 값이 아니라 메모리 주소를 가진 공간이며, 자료형은 그 공간을 어떤 규칙으로 해석할지를 결정한다.

기본 선언 형식 : 자료형 변수명 = 초기값;

예 : int count = 10;, double rate = 0.25;

변수명 규칙 : 알파벳, 숫자, 언더바(_) 사용 가능 / 숫자로 시작 불가 / 공백 사용 불가 / C 키워드 사용 불가 / 대소문자 구분

변수 선언과 초기화

int a;              // 선언 (값은 쓰레기 값)
int b = 10;        // 선언과 동시에 초기화
double c = 3.14;   // 실수형 변수

a = 5;              // 대입 (값 변경)

중요 : 초기화되지 않은 변수는 예측 불가능한 값을 가진다. C에서는 자동으로 0으로 초기화되지 않는다.

자료형(Data Type)의 역할

자료형은 메모리 크기뿐 아니라, 해석 방식(정수/실수/문자), 연산 규칙, 출력 형식까지 결정한다.

자료형 byte bit 범위(대표)
정수형(Integer)
short 2 16 -32,768 ~ 32,767
int 4 32 -2,147,483,648 ~ 2,147,483,647
long 4 32 컴파일러/플랫폼 의존
long long 8 64 -263 ~ 263-1
실수형(Floating Point)
float 4 32 약 7자리 정밀도
double 8 64 약 15자리 정밀도
문자형(Character)
char 1 8 -128 ~ 127 또는 0 ~ 255

주의 : 자료형의 크기와 범위는 C 표준 + 컴파일러/플랫폼에 따라 달라질 수 있다. 실무에서는 sizeof로 직접 확인하는 습관이 중요하다.

sizeof 연산자

sizeof는 자료형이나 값이 메모리에서 차지하는 바이트 크기를 반환한다. 함수가 아니라 컴파일 타임 연산자다. 

printf("%zu\n", sizeof(int));      // 4
printf("%zu\n", sizeof(double));   // 8

printf("%zu\n", sizeof(100));      // int 리터럴 → 4
printf("%zu\n", sizeof(3.14));     // double 리터럴 → 8

printf 서식 : sizeof 결과는 size_t 타입이므로 %zu 사용이 가장 안전하다.

문자(char)와 아스키 코드

C에서 char는 문자 타입이면서 동시에 1바이트 정수다. 문자는 내부적으로 숫자(아스키 코드)로 처리된다. 

char c = 'A';
int a = 65;

printf("%c\n", c);   // A
printf("%d\n", c);   // 65
printf("%c\n", a);   // A

문자 연산이 가능한 이유는 문자 = 숫자 구조이기 때문이다.

상수(Constant)

상수는 프로그램 실행 중 값이 변경되지 않도록 고정된 데이터다. 의도치 않은 변경을 막고, 코드의 의미를 명확히 하며, 유지보수를 쉽게 만든다.

상수를 사용하는 이유

• 안전성 : 실수로 값을 변경하는 버그를 컴파일 타임에 방지

• 가독성 : 매직 넘버(의미 없는 숫자) 대신 의미 있는 이름 사용

• 유지보수 : 값 변경 시 한 곳만 수정하면 전체에 반영

• 최적화 : 컴파일러가 상수를 인라인 처리하여 성능 향상 가능

const 키워드 (권장)

const는 변수 선언 시 앞에 붙여 해당 변수를 상수로 만든다. 타입을 가지므로 컴파일러의 타입 체크를 받고, 디버깅이 용이하다.

const 상수 기본 사용 
// 기본 상수 선언
const int MAX_SIZE = 100;
const float PI = 3.141592f;

const와 포인터

포인터와 const를 함께 사용할 때는 const의 위치에 따라 의미가 달라진다. 이 부분은 많은 초보자가 혼란스러워하는 부분이다.

const와 포인터의 조합 
int value = 10;
int other = 20;

// 1. 포인터가 가리키는 "값"을 변경 불가
const int* ptr1 = &value;
*ptr1 = 20;       // 오류! 값 변경 불가
ptr1 = &other;    // OK, 다른 주소 가리킬 수 있음

// 2. 포인터 "자체"를 변경 불가
int* const ptr2 = &value;
*ptr2 = 20;       // OK, 값 변경 가능
ptr2 = &other;    // 오류! 다른 주소 가리킬 수 없음

// 3. 둘 다 변경 불가
const int* const ptr3 = &value;
*ptr3 = 20;       // 오류!
ptr3 = &other;    // 오류!

const 위치 해석법 (오른쪽에서 왼쪽으로 읽기)

• const int* ptr : ptr은 "const int를 가리키는 포인터" → 값 변경 불가

• int* const ptr : ptr은 "int를 가리키는 const 포인터" → 주소 변경 불가

• const int* const ptr : 둘 다 변경 불가

#define 매크로 상수

#define은 전처리기 지시문으로, 컴파일 전에 텍스트를 단순 치환한다. 타입 개념이 없고 디버깅이 어려워 주의가 필요하다.

#define의 위험성 - 단순 치환 문제 
#define SQUARE(x) x * x

int main(void) {
    int a = 3;

    // 예상: 16, 실제: 7
    printf("%d\n", SQUARE(a + 1));  // a + 1 * a + 1 = 3 + 3 + 1 = 7

    // 올바른 매크로 정의
    #define SQUARE_SAFE(x) ((x) * (x))
    printf("%d\n", SQUARE_SAFE(a + 1)); // ((a + 1) * (a + 1)) = 16

    return 0;
}

enum을 이용한 상수 정의

enum(열거형)은 관련된 정수 상수들을 그룹으로 정의할 때 유용하다. 가독성이 좋고 타입 안전성도 제공한다.

// SUN=0, MON=1, TUE=2 ... 자동 할당

// 시작 값 지정

// JAN=1, FEB=2, MAR=3 ... 순차 증가

// 상태 코드 정의
enum Status {
    SUCCESS = 0,
    ERROR_FILE = -1,
    ERROR_MEMORY = -2,
    ERROR_INPUT = -3
};

// 사용 예

    printf("주말입니다!\n");

상수 사용 시 흔한 실수

• const 변수 미초기화 : const는 선언과 동시에 반드시 초기화해야 함

• 매크로 괄호 누락 : #define SQUARE(x) x*x 처럼 괄호 없이 정의하면 연산 순서 오류

• 매직 넘버 사용 : if (age > 18) 대신 if (age > MIN_ADULT_AGE) 권장

• const 포인터 혼동 : const int* 와 int* const의 차이 이해 필요

• 매크로 재정의 : 같은 이름의 매크로를 여러 번 정의하면 경고 발생

상수 활용 실전 예제 
#include <stdio.h>

// 프로그램 전역 설정값

// 성적 등급

char getGrade(double score) {
    if (score >= A) return 'A';
    if (score >= B) return 'B';
    if (score >= C) return 'C';
    if (score >= D) return 'D';
    return 'F';
}

int main(void) {
    double score = 85.5;

    printf("[%s] 성적 결과\n", SCHOOL_NAME);
    printf("점수: %.1f점 / 등급: %c\n", score, getGrade(score));
    printf("합격 여부: %s\n", score >= PASS_SCORE ? "합격" : "불합격");

    return 0;
}

리터럴(Literal)

리터럴은 코드에 직접 작성된 값 자체를 의미한다. 변수에 저장되기 전의 "원시 데이터"로, 컴파일러가 자동으로 타입을 추론한다.

리터럴 vs 변수 vs 상수

• 리터럴 : 코드에 직접 쓴 값 (예: 10, 3.14, 'A')

• 변수 : 값을 저장하는 메모리 공간 (값 변경 가능)

• 상수 : 변경 불가능한 변수 (const int MAX = 100;)

실수 리터럴

실수 리터럴은 기본적으로 double 타입이다. float로 사용하려면 f 접미사를 붙여야 한다.

실수 리터럴

double d = .5;        // 0.5 (정수부 생략 가능)

long double g = 3.14L; // long double (L 접미사)

• float x = 3.14f; → 올바른 사용법

    // 2) 오름차순 정렬(선택 정렬)
    for (int i=0; i<k-1; i++) {
        int minIdx = i;
        for (int j=i+1; j<k; j++) {
            if (out[j] < out[minIdx]) minIdx = j;
        }
        if (minIdx != i) {
            int tmp = out[i];
            out[i] = out[minIdx];
            out[minIdx] = tmp;
        }
    }

    // 3) 출력
    printf("[");
    for (int i=0; i<k; i++) {
        printf("%d", out[i]);
        if (i != k-1) printf(", ");
    }
    printf("]\n");

    return 0;
}

정답 예시 (배열/문자열 기반, 동적할당 없이)
// 심화: 토큰화 + 정규화 + 카운팅 + Top-K
// 입력 예: "Hello, HELLO... c99 and C99; hi! a b 007 007."
// 출력: hello(2), c99(2), 007(2), and(1) ... (Top 5)
// 제한: 동적할당 없이 배열로만 처리

#include <stdio.h>
#include <string.h>

#define MAX_LINE 500
#define MAX_WORDS 200
#define MAX_LEN 32
#define TOPK 5

int is_alnum(char ch) {
    return (ch >= '0' && ch <= '9') ||
           (ch >= 'a' && ch <= 'z') ||
           (ch >= 'A' && ch <= 'Z');
}

char to_lower(char ch) {
    if (ch >= 'A' && ch <= 'Z') return ch - 'A' + 'a';
    return ch;
}

// words[i] : i번째 단어, cnt[i] : 빈도, used : 저장된 단어 개수
int find_word(char words[][MAX_LEN], int used, const char* w) {
    for (int i=0; i<used; i++) {
        if (strcmp(words[i], w) == 0) return i;
    }
    return -1;
}

// 정렬: 빈도 내림차순, 빈도 같으면 사전순 오름차순
void sort_words(char words[][MAX_LEN], int cnt[], int used) {
    for (int i=0; i<used-1; i++) {
        for (int j=0; j<used-1-i; j++) {
            int swap = 0;
            if (cnt[j] < cnt[j+1]) swap = 1;
            else if (cnt[j] == cnt[j+1] && strcmp(words[j], words[j+1]) > 0) swap = 1;

            if (swap) {
                int tc = cnt[j]; cnt[j] = cnt[j+1]; cnt[j+1] = tc;
                char tmp[MAX_LEN];
                strcpy(tmp, words[j]);
                strcpy(words[j], words[j+1]);
                strcpy(words[j+1], tmp);
            }
        }
    }
}

int main() {
    char line[MAX_LINE];
    char words[MAX_WORDS][MAX_LEN] = {""};
    int cnt[MAX_WORDS] = {0};
    int used = 0;

    // 한 줄 입력(공백 포함)
    scanf("%499[^\n]", line);

    char token[MAX_LEN];
    int t = 0;

    // 문자열을 1글자씩 훑으면서 토큰 생성
    for (int i=0; ; i++) {
        char ch = line[i];

        // 끝(널)도 구분자로 취급해서 마지막 토큰 flush
        if (ch == '\0' || !is_alnum(ch)) {
            if (t > 0) {
                token[t] = '\0';

                // 길이 1 토큰 제외
                if (t > 1) {
                    int idx = find_word(words, used, token);
                    if (idx >= 0) {
                        cnt[idx]++;
                    } else {
                        if (used < MAX_WORDS) {
                            strcpy(words[used], token);
                            cnt[used] = 1;
                            used++;
                        }
                    }
                }
                t = 0;
            }

            if (ch == '\0') break;
        } else {
            // 토큰에 추가(소문자 정규화)
            if (t < MAX_LEN-1) {
                token[t++] = to_lower(ch);
            } else {
                // 너무 긴 토큰은 잘라서 저장(버퍼 보호)
                // 여기서는 단순히 이후 문자를 무시하고 계속 진행
            }
        }
    }

    sort_words(words, cnt, used);

    printf("=== Top %d ===\n", TOPK);
    int limit = (used < TOPK) ? used : TOPK;
    for (int i=0; i<limit; i++) {
        printf("%d) %s (%d)\n", i+1, words[i], cnt[i]);
    }

    return 0;
}

int ex2_sum_odd_reverse(const int* p, int n) {
    if (!p || n<=0) return 0;

    int sum = 0;
    const int* cur = p + (n - 1);

    for (int i=0; i<n; i++, cur--) {
        if ((*cur & 1) != 0) sum += *cur;
    }
    return sum;
}
EXERCISE 3 — 두 포인터로 배열 역순 뒤집기

목표 : 양 끝에서 시작하는 두 포인터를 안쪽으로 좁히며 swap합니다.

루프 조건 : left < right 동안만 swap을 수행합니다. 같아지는 순간(중앙)은 종료 지점입니다.

장점 : 추가 배열 없이 O(1) 메모리로 처리합니다.

풀이 코드
void ex3_reverse_inplace(int* p, int n) {
    if (!p || n<=1) return;

    int* left = p;
    int* right = p + (n - 1);

    while (left < right) {
        int tmp = *left;
        *left = *right;
        *right = tmp;
        left++;
        right--;
    }
}

Call-by-Value / Call-by-Reference

Call-by-Value (값에 의한 호출) : 함수 호출 시 인자의 값을 복사하여 전달. 함수 내에서 매개변수를 변경해도 원본에 영향 없음.

Call-by-Reference (참조에 의한 호출) : 함수 호출 시 인자의 주소를 전달. 함수 내에서 포인터로 원본 값을 직접 수정 가능.

두 방식 비교

구분 Call-by-Value Call-by-Reference
전달 방식 값 복사 주소 전달
매개변수 일반 변수 포인터 변수
호출 방식 func(a) func(&a)
원본 변경 불가능 가능
사용 예 계산 결과 반환 값 교환, 여러 값 반환

재귀호출 함수 (Recursion)

재귀함수(Recursive Function) : 함수 내부에서 자기 자신을 호출하는 함수

필수 조건 : (1) 종료 조건(Base Case) 필수 (2) 호출마다 종료 조건에 가까워져야 함

주의 : 종료 조건 없으면 무한 호출로 스택 오버플로우 발생

재귀 vs 반복문

구분 재귀 반복문
구조 자기 자신 호출 for, while 반복
메모리 호출마다 스택 사용 고정 메모리
가독성 수학적 정의와 유사 직관적
성능 느릴 수 있음 일반적으로 빠름
적합한 문제 트리, 분할정복 단순 반복 작업

재귀함수 기본 구조

재귀 기본 패턴
반환형 함수명(매개변수) {
    // 1. 종료 조건 (Base Case)
    if (종료조건) {
        return 기본값;
    }

    // 2. 재귀 호출 (Recursive Case)

int factorialLoop(int n) {

    int t = *a; *a = *b; *b = t;

void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(&arr[j], &arr[j + 1]);
            }
        }
    }
}

int main() {
    int nums[] = {5, 2, 8, 1, 9};
    bubbleSort(nums, 5);
    for (int i = 0; i < 5; i++) printf("%d ", nums[i]);
    // 1 2 5 8 9
    return 0;
}

#include <time.h>

int generateNumber(int min, int max) {

int checkGuess(int guess, int answer) {
    if (guess > answer) {
        printf("DOWN! 더 작은 수입니다.\n");
        return 0;
    } else if (guess < answer) {
        printf("UP! 더 큰 수입니다.\n");
        return 0;
    }
    return 1;
}

    int answer = generateNumber(1, 100);

        int guess = getInput();
        count++;

        }
    }
