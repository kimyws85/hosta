### UX/UI (4 Parts / 16 Topics)

**Part 1. UX 기초**
- UX란 무엇인가
  - UX의 정의와 범위 (사용 전/중/후 경험, 인터랙션, 정보 구조, 콘텐츠, 성능, 접근성)
  - UX와 UI의 차이 (범위, 초점, 산출물, 비유)
  - UX 디자이너의 역할 (리서치, 정보 구조, 유저 플로우, 프로토타입, 테스트)
  - 좋은 UX vs 나쁜 UX 사례 비교
  - UX가 중요한 이유 (비즈니스 성과, 비용 절감, 경쟁 우위)
- 사용자 이해
  - 사용자 중심 디자인(UCD) 원칙 4가지
  - 페르소나 작성 방법 (기본 정보, 배경, 목표, 페인 포인트, 기술 수준, 인용문)
  - 공감 지도(Empathy Map) 활용 (Think, Feel, Say, Do, Pain, Gain)
  - 사용자 시나리오 작성 (상황, 행동, 기대)
  - 사용자 세그먼테이션 (빈도, 목표, 기술 수준, 동기)
- UX 피라미드
  - UX 허니콤 7가지 (Useful, Usable, Desirable, Findable, Accessible, Credible, Valuable)
  - UX 계층 구조 5단계 (기능성 → 신뢰성 → 사용성 → 편의성 → 즐거움)
  - 각 단계별 상세 설명 및 체크리스트
  - 실전 적용: 이커머스 결제 프로세스 단계별 개선 사례
- 심리학과 디자인 ⭐ 대폭 확장
  - **인지 심리학 개요**
    - 뇌의 정보 처리 방식 (매초 1100만 비트 → 50비트 의식 처리)
    - 사용자 행동 패턴 연구 데이터 (첫 인상 0.05초, 스캐닝 패턴)
    - 심리학 원리 적용 시 전환율 20~30% 향상 (Baymard Institute)
  - **게슈탈트 원리 (Gestalt Principles)** - 6가지 원리 심화
    - 근접성(Proximity): 관련 요소 그룹핑, 여백 활용 전략, Airbnb/Stripe 사례
    - 유사성(Similarity): 시각적 일관성, 버튼 계층 구조, GitHub 사례
    - 폐쇄성(Closure): 단순화 전략, 로고 디자인, FedEx/Apple 사례
    - 연속성(Continuity): 시선 흐름 설계, Z/F 패턴, Amazon 체크아웃 사례
    - 공동 운명(Common Fate): 애니메이션 그룹핑, Trello/iOS 사례
    - 배경-전경(Figure-Ground): 대비 설계, 모달 처리, Medium/Slack 사례
    - 원리별 체크포인트 비교표
  - **힉의 법칙 (Hick's Law)** - 선택지와 의사결정 시간
    - 공식: T = a + b × log₂(n + 1)
    - 연구 데이터: 24종 vs 6종 잼 실험 (구매율 3% vs 30%), Amazon/Google/Netflix 연구
    - 적용 1: 메뉴 단순화 (5~7개 카테고리, Apple.com/Notion 사례)
    - 적용 2: 단계별 정보 제공 (멀티스텝 폼, Typeform/Shopify 온보딩 사례)
    - 적용 3: 추천과 기본값 (Mailchimp 요금제, Netflix 추천 사례)
    - 주의사항: 너무 단순하면 안 됨 (명확한 카테고리 필요)
    - 실전 비교표 (회원가입, 검색 필터, 상품 정렬, 결제 수단, 알림 설정)
  - **피츠의 법칙 (Fitts's Law)** - 타겟 크기와 거리
    - 공식: T = a + b × log₂(D/W + 1)
    - 연구 데이터: Microsoft(오클릭률 62% 감소), Apple/Google 가이드라인
    - 적용 1: 중요 버튼 크게 (44x44px/48x48px, Spotify/Amazon 사례)
    - 적용 2: 자주 쓰는 기능 가깝게 (Gmail/Figma 사례)
    - 적용 3: 화면 모서리 활용 (무한한 크기, macOS 메뉴바/Windows 시작 버튼)
    - 적용 4: 작업 흐름 최적화 (Dropbox/Twitter 사례)
    - 모바일 엄지 영역(Thumb Zone) 상세 가이드 (Steven Hoober 연구, Instagram/iOS Safari 사례)
    - 6개 상황별 비교표 (크기/위치/영향)
  - **밀러의 법칙 (Miller's Law)** - 작업 기억 용량
    - 7±2 원리 (현대 연구: 4±1개가 더 정확)
    - 청킹(Chunking) 전략: 전화번호/카드번호 분할 (Stripe 사례)
    - 적용 1: 메뉴 항목 5~7개 제한 (Apple.com/Slack 사례)
    - 적용 2: 프로세스 3~5단계 제한 (Airbnb/Uber 사례)
    - 적용 3: 대시보드 위젯 제한 (Google Analytics/Salesforce 사례)
    - 적용 4: 폼 필드 그룹핑 (Amazon 체크아웃/LinkedIn 프로필)
    - 인지 부하(Cognitive Load) 관리 전략 (내재적/외재적/본유적 부하)
  - **제이콥의 법칙 (Jakob's Law)** - 사용자 기대와 관습
    - 핵심 개념: 사용자는 익숙한 패턴 기대 (Nielsen Norman Group 연구)
    - 보편적 UI 관습 8가지 (로고, 햄버거 메뉴, 장바구니, 검색, 링크, 버튼, 모달, 페이지네이션)
    - 업계별 표준 패턴 (이커머스, SNS, 스트리밍, 뱅킹, SaaS, 뉴스/블로그)
    - 혁신 vs 관습: 따라야 할 것 vs 차별화 가능한 것
    - 멘탈 모델 활용 (휴지통, 폴더, 장바구니, 탭, 알림)
    - 점진적 혁신 전략 (Instagram Stories, YouTube Shorts, Gmail Tabs 사례)
    - 업계별 베스트 프랙티스 비교표
  - **추가 인지 심리학 법칙 10가지**
    - 시리얼 포지션 효과(Serial Position Effect): 첫/끝 항목 기억 (Ebbinghaus 연구)
    - 폰 레스토프 효과(Von Restorff Effect): 다른 것이 눈에 띔 (격리 효과)
    - 자이가르닉 효과(Zeigarnik Effect): 미완성 작업 기억 (진행률 표시)
    - 피크-엔드 법칙(Peak-End Rule): 절정과 마지막 순간 기억 (Kahneman 연구)
    - 미적 사용성 효과(Aesthetic-Usability Effect): 아름다움=쉬움 인식 (Kurosu & Kashimura)
    - 목표 그래디언트 효과(Goal-Gradient Effect): 목표 가까울수록 동기↑ (Kivetz 연구)
    - 테슬러의 복잡도 보존 법칙(Tesler's Law): 복잡도는 이동할 뿐
    - 파킨슨의 법칙(Parkinson's Law): 시간만큼 작업 확장 (긴급성 활용)
    - 포스텔의 법칙(Postel's Law): 입력엔 관대, 출력엔 엄격
    - 도허티 임계값(Doherty Threshold): <400ms 응답 시간의 중요성
  - **상황별 적용 가이드** - 10가지 디자인 상황별 법칙 매핑표
  - **종합 체크리스트** - 26개 항목 검증 리스트 (게슈탈트 5개, 힉 3개, 피츠 3개, 밀러 3개, 제이콥 3개, 기타 9개)
  - **핵심 요약** - 5가지 핵심 원칙 + 실전 적용 유의사항
  - 실제 기업 사례 50+ (Google, Apple, Amazon, Airbnb, Stripe, GitHub, Netflix, Spotify 등)

**Part 2. 리서치와 분석**
- 문제 정의
  - 문제 정의의 중요성 (해결 방향, 이해관계자 정렬, 범위 한정, 성공 측정)

  - 인터뷰 설계 4단계 (목표 정의, 대상 선정, 질문 준비, 진행 및 분석)
  - 인터뷰 스크립트 예시 (오픈 질문, 구체적 질문, 감정 질문, 행동 질문)
  - 설문조사 설계 (명확한 목표, 간결한 질문, 척도 일관성, 유도 질문 지양)
  - 맥락적 관찰 방법 (Think Aloud, Shadowing, Diary Study)
  - 어피니티 다이어그램 (데이터 그룹핑 → 테마 도출 → 인사이트 발견)
- 사용자 여정 맵
  - 여정 맵 9가지 구성요소 (페르소나, 시나리오, 단계, 행동, 터치포인트, 생각/감정, 페인포인트, 기회, 소유자)
  - 이커머스 쇼핑 여정 예시 (인지 → 탐색 → 비교 → 구매 → 배송 → 사후)
  - 감정 곡선 그리기 (Y축: 만족도, X축: 시간/단계)
  - 페인 포인트 우선순위화 공식 (심각도 × 빈도 × 임팩트)
  - 개선 기회 도출 프로세스 (페인 포인트 분석 → Quick Win 찾기 → 솔루션 아이디어)
  - 여정 맵 작성 체크리스트
- 경쟁 분석
  - 경쟁 분석의 목적 (시장 표준 이해, 베스트 프랙티스, 차별화 포인트, 기회/위협)
  - 경쟁사 유형 4가지 (직접, 간접, 잠재적, 영감 경쟁사)
  - 기능 비교 매트릭스 (핵심 기능 나열 → 제공 여부 표시 → 차별점 발견)
