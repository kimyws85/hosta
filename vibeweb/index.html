기존 코딩 학원과 무엇이 다른가요?

단순히 코드를 외우는 수업이 아닙니다. AI 시대에 맞는 방식으로 배웁니다.

AI 기반 웹 제작 환경 구축

Claude + 웹디자인 기초 + 프롬프트 웹 제작

코딩을 전혀 몰라도 괜찮습니다. 이 달의 목표는 Claude라는 AI 도구를 완벽히 세팅하고, 프롬프트(말로 하는 명령)만으로 실제 웹사이트를 만들어보는 경험을 하는 것입니다. 디자인 감각의 기초도 함께 잡아갑니다.

웹디자인 기초

  • 웹의 구조 이해 — 모든 웹사이트는 헤더(상단), 본문(내용), 푸터(하단)로 이루어집니다. 좋은 웹사이트의 레이아웃이 어떻게 구성되는지 실제 사례로 분석합니다.
  • 색상 이론 기초 — 주색(brand color), 보조색(accent), 배경색, 텍스트 색의 역할을 이해합니다. 색상 대비(contrast)가 가독성에 미치는 영향, 어두운 배경 vs 밝은 배경의 특성을 배웁니다.
  • 폰트와 타이포그래피 — 제목/소제목/본문/캡션의 크기 위계(hierarchy)를 잡는 법을 익힙니다. 가독성 좋은 폰트 선택 기준, 줄 간격(line-height)이 읽힘에 미치는 영향을 이해합니다.
  • 여백(Spacing)의 중요성 — 요소들 사이의 간격이 왜 디자인 품질을 결정하는지 이해합니다. 여백이 없는 사이트 vs 여백이 적절한 사이트를 직접 비교 분석합니다.
  • 좋은 웹 vs 나쁜 웹 분석 — 실제 국내외 사이트들을 보며 "왜 이게 좋은가", "왜 이게 별로인가"를 색상·폰트·여백·구조 관점에서 설명할 수 있는 눈을 기릅니다.

Claude + VSCode 세팅

  • Claude Pro 가입 — Claude.ai 유료 플랜(Pro) 가입 방법, 플랜별 차이점, 무료 vs 유료를 왜 유료로 써야 하는지 이유를 설명합니다.
  • VSCode 설치 및 세팅 — 개발자들이 가장 많이 쓰는 에디터 VSCode를 설치하고, 기본 테마, 폰트 크기, 자동 저장 설정 등 편한 환경을 만듭니다.
  • Claude Code (CLI) 설치 — 터미널/명령줄에서 Claude를 사용할 수 있는 Claude Code를 설치합니다. 왜 웹 인터페이스보다 CLI(Command Line Interface)가 개발에 더 강력한지 이해합니다.
  • VSCode + Claude Code 연동 — VSCode 안에서 바로 Claude와 대화하면서 코드를 만들고 수정하는 실제 개발 환경을 완성합니다. 파일을 열고 바로 작업하는 워크플로우를 익힙니다.
  • 기본 사용법 실습 — 파일 열기, 폴더 구조 이해, Claude에게 질문하는 법, Claude가 만든 파일을 브라우저에서 확인하는 법까지 실습합니다.

프롬프트로 웹 만들기

  • UI 컴포넌트 단위로 만들기 — 버튼(크기·색상·hover 효과), 카드(이미지+텍스트 조합), 슬라이더/캐러셀, 아이콘 배치, 네비게이션 바 등 웹에서 자주 쓰이는 요소들을 하나씩 Claude에게 만들어달라고 요청하며 경험을 쌓습니다.
  • 프롬프트 수정 요청하기 — "버튼 색상을 파란색으로 바꿔줘", "글자 크기 키워줘", "카드 간격 더 넓게 해줘" 등 원하는 결과를 말로 요청하고 수정하는 반복 과정을 익힙니다.
  • 자기소개 페이지 완성 — 이름, 소개글, 프로필 이미지, SNS 링크, 기술 스택 등이 들어간 실제 포트폴리오용 자기소개 페이지를 Claude와 함께 완성합니다.
  • 랜딩 페이지 완성 — 히어로 섹션(메인 문구+이미지), 특징 소개(카드 3개), 후기/수치 섹션, CTA 버튼(행동 유도)이 포함된 실제 서비스 랜딩 페이지를 만듭니다.
1개월 후 결과물 자기소개 페이지 1개 + 랜딩 페이지 1개 완성. 코드를 한 줄도 직접 안 써도 됩니다.

웹 표준 기술 이해

HTML / CSS / JS + Claude

1개월차에서 프롬프트로만 만들었다면, 이번 달은 Claude가 만든 코드를 직접 읽고 이해합니다. HTML, CSS, JavaScript의 기본 구조를 파악하면 "왜 이렇게 보이는지"를 알게 되고, 원하는 부분을 직접 수정할 수 있는 능력이 생깁니다.

HTML — 웹의 뼈대

  • 태그 구조 이해 — <div>, <section>, <p>, <h1~h6>, <img>, <a>, <ul><li>, <span> 등 자주 쓰이는 태그들의 역할과 사용법을 이해합니다.
  • 시맨틱 태그 — <header>, <nav>, <main>, <section>, <article>, <aside>, <footer> 등 의미있는 태그를 쓰는 이유(검색엔진 최적화, 접근성)를 이해하고 적용합니다.
  • HTML 파일 구조 — <!DOCTYPE html>, <head>, <body>의 역할, CSS/JS 파일을 연결하는 방법, meta 태그의 역할을 이해합니다.

CSS — 웹의 스타일

  • 선택자·클래스·id 개념 — CSS가 어떻게 특정 요소를 찾아서 스타일을 적용하는지 이해합니다. 클래스(.class)와 id(#id)의 차이와 언제 무엇을 쓰는지 배웁니다.
  • CSS 변수(Custom Properties) — :root에 색상, 폰트, 간격을 변수로 정의하고 전체 스타일을 한 곳에서 관리하는 실무 방식을 익힙니다.
  • Flexbox 레이아웃 — 요소를 가로/세로로 정렬하는 Flexbox의 핵심 속성들 (display:flex, justify-content, align-items, gap 등)을 실습합니다.
  • Grid 레이아웃 — 카드 목록, 2단 레이아웃 등을 Grid로 구현하는 법을 배웁니다. Flexbox와 Grid를 언제 각각 쓰는지 판단하는 기준도 익힙니다.
  • 반응형 디자인 (미디어 쿼리) — 화면 크기에 따라 레이아웃이 바뀌는 반응형 웹을 만드는 법을 배웁니다. 모바일(480px), 태블릿(768px), 데스크톱 기준으로 스타일을 분기하는 법을 익힙니다.
  • transition · hover 효과 — 버튼을 올렸을 때 색이 바뀌고, 카드를 올렸을 때 살짝 올라오는 등의 부드러운 인터랙션 효과를 CSS만으로 구현하는 법을 익힙니다.

JavaScript — 웹의 동작

  • 변수·함수 기초 — let, const, 함수 선언, 화살표 함수 등 JS의 기본 문법을 이해합니다. 코드를 외우는 것이 목표가 아니라, Claude가 만든 코드를 읽고 이해하는 것이 목표입니다.
  • DOM 조작 기초 — document.querySelector로 요소를 찾고, classList.add/remove로 클래스를 추가/제거하는 법을 이해합니다. 이것만 알면 대부분의 인터랙션을 구현할 수 있습니다.
  • 이벤트 리스너 — 클릭, 스크롤, 마우스 이동 등의 이벤트에 반응하는 addEventListener의 기본 패턴을 익힙니다.
  • 실습: 모달 팝업 — 버튼 클릭 시 팝업창이 뜨고, X 클릭 시 닫히는 모달을 직접 구현합니다.
  • 실습: 모바일 햄버거 메뉴 — 모바일에서 햄버거 메뉴 클릭 시 메뉴가 펼쳐지는 토글 네비게이션을 구현합니다.
  • 실습: 스크롤 애니메이션 — 스크롤 시 요소가 부드럽게 나타나는 IntersectionObserver 패턴을 익힙니다.

Claude와 함께하는 방식

  • Claude 코드 읽고 이해하기 — Claude가 만든 코드를 보면서 "이 부분이 왜 이렇게 된 건지"를 Claude에게 물어보고 설명을 들으며 코드 이해력을 키웁니다.
  • 직접 수정 + Claude 요청 병행 — 간단한 수정(색상 변경, 텍스트 수정)은 직접 하고, 복잡한 기능은 Claude에게 요청하는 방식으로 실력을 쌓습니다.
  • 1개월차 결과물 코드로 재구현 — 1개월차에 프롬프트만으로 만든 자기소개 페이지를 이번에는 HTML/CSS 구조를 직접 이해하며 다시 만들어봅니다.
2개월 후 결과물 반응형 웹사이트 1개 완성. HTML/CSS/JS 구조를 읽고 이해할 수 있는 수준. 간단한 부분은 직접 수정 가능.

컴포넌트 기반 개발

React + Claude

React는 현재 프론트엔드 취업 시장의 표준입니다. 복잡한 웹 앱을 "컴포넌트"라는 단위로 나눠서 만드는 방식을 배웁니다. Claude와 함께라면 React의 진입장벽을 낮추고 실무 수준의 앱을 빠르게 완성할 수 있습니다.

React 핵심 개념

  • 왜 React인가 — HTML로만 만들면 같은 코드를 여러 곳에 반복 작성해야 합니다. React는 버튼, 카드, 네비게이션 같은 요소를 "컴포넌트"로 한 번만 만들고 어디서든 재사용할 수 있게 해줍니다. 현재 국내외 대부분의 웹 서비스가 React로 만들어집니다.
  • 컴포넌트 개념 이해 — Button.jsx, Card.jsx, Navbar.jsx처럼 UI를 독립적인 조각으로 나누는 법, 컴포넌트를 조합해서 페이지를 만드는 방식을 이해합니다.
  • JSX 문법 — HTML처럼 생겼지만 JavaScript인 JSX 문법을 이해합니다. className, onClick, {} 안에 JS 표현식 넣기 등 HTML과 다른 점을 익힙니다.
  • Props — 컴포넌트에 데이터 전달 — 같은 Card 컴포넌트에 제목, 내용, 이미지를 다르게 넣어서 여러 카드를 만드는 props 개념을 이해하고 실습합니다.
  • State — 변하는 데이터 관리 — 버튼 클릭 횟수, 모달 열림/닫힘, 탭 선택 상태 등 화면에서 변하는 값을 useState로 관리하는 법을 익힙니다.
  • 파일 구조 이해 — src/, components/, pages/, public/ 폴더의 역할을 이해하고 실무에서 파일을 어떻게 구조화하는지 패턴을 익힙니다.

Claude와 React 개발

  • 컴포넌트 생성 요청 — "ProductCard 컴포넌트를 만들어줘. props로 image, title, price, description을 받아야 해" 처럼 구체적으로 요청해서 바로 쓸 수 있는 컴포넌트를 만드는 법을 익힙니다.
  • 기능 구현 요청 — "탭을 클릭하면 해당 탭의 내용만 보여주는 기능 만들어줘", "검색어를 입력하면 목록이 필터링되는 기능 만들어줘" 등의 실무 기능을 Claude와 함께 구현합니다.
  • 에러 해결 — React 개발 중 만나는 에러 메시지를 Claude에게 붙여넣고 원인과 해결법을 이해하는 방식으로 디버깅 능력을 키웁니다.

최종 프로젝트: 실무형 앱 완성

  • 프로젝트 선택 — 쇼핑몰 상품 목록 페이지 / 포트폴리오 사이트 / 레스토랑 메뉴 앱 중 하나를 선택합니다.
  • 컴포넌트 설계 — 만들 앱을 어떤 컴포넌트로 나눌지 설계하고, Claude와 함께 하나씩 만들어갑니다.
  • 기능 구현 — 필터링, 검색, 탭 전환, 모달 등 실제 서비스에서 볼 수 있는 기능들을 구현합니다.
  • 배포 — Vercel을 통해 실제 인터넷에서 접속 가능한 URL로 배포하는 방법을 배웁니다. 이제 포트폴리오에 링크를 넣을 수 있습니다.

서비스 설계 역량 강화

UI/UX + 기획 + Figma

개발만 할 줄 아는 사람과 기획부터 설계까지 할 수 있는 사람의 차이는 큽니다. 4개월차에는 Figma로 와이어프레임·프로토타입을 만들고, 서비스 기획서 작성법을 익히며 "AI 활용 기획자" 방향으로 커리어를 확장합니다.

UX 리서치 & 분석

  • 사용자 인터뷰 방법 — 사용자가 서비스를 어떻게 쓰는지 파악하기 위한 인터뷰 질문을 설계하고 진행하는 방법을 익힙니다. Claude로 인터뷰 스크립트를 작성하는 법도 배웁니다.
  • 페르소나 작성 — 서비스의 주요 사용자를 구체적으로 묘사한 페르소나 카드를 작성합니다. "28세 직장인 김민지, 퇴근 후 자기계발에 관심 있음" 같은 구체적 인물을 설정하고 그 사람의 니즈에 맞게 서비스를 설계하는 방법을 배웁니다.
  • 경쟁사 분석 — 비슷한 서비스들을 비교 분석하여 우리 서비스의 차별점을 찾는 경쟁사 분석 템플릿을 Claude로 작성하고 직접 채워봅니다.

정보구조 & 사용자 흐름 설계

  • IA(Information Architecture) 설계 — 서비스의 메뉴 구조, 페이지 간 연결 관계를 트리 형태로 설계합니다. "홈 → 상품 목록 → 상품 상세 → 장바구니 → 결제"처럼 전체 구조를 한눈에 그립니다.
  • User Flow 작성 — 사용자가 목표를 달성하기까지의 경로를 플로우 차트로 그립니다. "회원가입 → 로그인 → 상품 검색 → 구매" 과정에서 어디서 이탈하는지 파악합니다.

Figma 와이어프레임 & 프로토타입

  • Figma 기초 사용법 — 프레임, 도형, 텍스트, 컴포넌트, 오토 레이아웃 등 와이어프레임 제작에 필요한 Figma의 핵심 기능을 익힙니다.
  • 로우파이 와이어프레임 — 색상 없이 레이아웃과 정보 구조만 표현한 와이어프레임을 빠르게 만들어 방향성을 팀원/클라이언트와 확인하는 법을 배웁니다.
  • 하이파이 프로토타입 — 실제 색상·폰트·이미지가 들어간 완성도 높은 디자인을 Figma로 만들고, 클릭하면 페이지가 이동하는 인터랙티브 프로토타입을 제작합니다. 링크를 공유하면 누구나 앱처럼 체험할 수 있습니다.
  • 디자인 시스템 구성 — 프로젝트 전체에서 쓸 컬러 팔레트, 타이포그래피, 버튼·카드 컴포넌트를 Figma에서 정의하고 일관성 있게 관리하는 법을 익힙니다.
  • 사용성 평가 — 프로토타입을 실제 사람에게 써보게 하며 어디서 막히는지 찾아내는 간단한 사용성 테스트 방법을 배웁니다.

서비스 기획

  • 기능 명세서(요구사항 정의) — 개발자에게 전달할 수 있는 수준의 기능 명세서를 작성합니다. "회원가입 버튼 클릭 시 — 이메일 형식 검증 → 중복 확인 → 완료 알림"처럼 구체적인 동작 흐름을 문서화하는 법을 익힙니다.
  • 개발자·디자이너와 협업하는 법 — 개발자가 이해할 수 있는 언어로 요구사항을 전달하는 법, 피그마 파일을 개발자에게 넘기는 방식, 피드백을 주고받는 방법을 배웁니다.
  • Claude로 기획 문서 작성 — 기획서 초안, 사용자 스토리, 기능 명세서를 Claude와 함께 빠르게 작성하는 실무 프롬프트 패턴을 익힙니다.

일반 직업도 위험하다

"내 직업은 창의적이라서 괜찮다"는 생각은 위험합니다. 단순 반복뿐 아니라 지식 기반 직업도 빠르게 변화하고 있습니다.

  • 회계 담당자의 하루치 엑셀 작업 → AI가 5분 만에 처리
  • 번역가의 하루 번역량 → AI가 1초 만에 완성
  • 디자인 초안 작업 → AI가 10초 만에 시안 10개 생성
  • 고객센터 상담원 → AI 챗봇이 24시간 무인 운영
  • 법률 문서 검토 → AI가 수백 페이지를 몇 분 만에 분석
  • 콘텐츠 작성 · 보고서 초안 → AI가 즉시 초안 생성

개발자도 예외가 아니다

"개발자는 AI로 대체 안 되겠지"라는 생각은 가장 위험한 착각입니다. 코드만 짜는 개발자는 이미 위험합니다.

  • 반복적인 CRUD 구현 → AI가 더 빠르고 정확하게 처리
  • 버그 찾기 · 수정 → AI가 코드 전체를 분석해 즉시 제안
  • 단순 API 연동 작업 → AI가 코드 한 번에 완성
  • AI 활용 개발자 1명 = 일반 개발자 3~5명의 생산성
  • 실제로 IT 기업들이 채용 공고에 "AI 활용 능력" 필수 추가 중
  • 살아남는 개발자 = AI로 더 큰 문제를 해결하는 개발자

AI를 먼저 배운 사람의 기회

두려워하기보다 먼저 배우는 것이 답입니다. AI를 이해하는 사람에게는 엄청난 레버리지가 생깁니다.

  • AI 활용자 1명이 팀 전체의 생산성을 혼자 냄
  • 코드를 전혀 몰라도 AI로 실제 서비스를 만드는 시대
  • 기획자가 직접 프로토타입을 만들어 개발팀과 대화 가능
  • 더 적은 비용으로 더 빠르게 사이드 프로젝트 완성 가능
  • AI를 모르는 경쟁자보다 항상 앞서 나갈 수 있는 구조
  • AI는 위기가 아닌, 먼저 잡으면 초강력 무기

AI(LLM)는 어떻게 작동하나요? — 멘토가 꼭 알아야 할 기초 지식

수강생이 "AI가 뭔가요?" 라고 물었을 때 자신있게 설명할 수 있어야 합니다. 기술적 깊이보다 핵심 개념을 쉽게 전달하는 게 목표입니다.

PART 01

LLM이 뭔가요? — 대형 언어 모델의 기본 개념

LLM이란?

Large Language Model, 즉 "대형 언어 모델"의 줄임말입니다. 인터넷, 책, 코드, 논문 등 수천억 개의 텍스트 데이터를 학습한 AI입니다.

  • GPT-4o (OpenAI), Claude (Anthropic), Gemini (Google)이 대표적
  • 우리가 사용하는 Claude도 LLM의 한 종류
  • 단어를 하나씩 예측하며 문장을 완성하는 방식으로 작동
어떻게 학습하나요?

LLM은 3단계 과정을 통해 만들어집니다. 이 과정을 이해하면 왜 AI가 이런 특성을 가지는지 설명할 수 있습니다.

  • 사전학습(Pre-training) — 수천억 텍스트로 언어 자체를 이해
  • 파인튜닝(Fine-tuning) — 대화·코드·지시 따르기 특화 훈련
  • RLHF — 사람이 좋은 답과 나쁜 답을 평가해 AI를 개선
토큰이란?

AI는 텍스트를 "토큰" 단위로 처리합니다. 단어 하나가 1~3개의 토큰이 됩니다. 이것이 AI 사용 비용과 직결됩니다.

  • 영어 1단어 ≈ 1.3 토큰 / 한글 1단어 ≈ 2~4 토큰
  • Claude Pro: 200K 토큰 컨텍스트 (약 15만 단어 분량)
  • 대화가 길어질수록 더 많은 토큰을 소비
컨텍스트 윈도우란?

AI가 한 번에 기억할 수 있는 텍스트의 양입니다. 이 범위를 벗어나면 AI는 앞의 내용을 잊어버립니다.

  • Claude의 강점 중 하나가 매우 긴 컨텍스트 윈도우
  • 코드 전체를 붙여넣고 수정 요청해도 정확하게 처리 가능
  • 긴 대화를 해도 초반 내용을 기억하는 이유
PART 02

내가 질문하면 AI 안에서 무슨 일이 벌어지나요?

수강생이 "AI가 어떻게 이걸 알아요?"라고 물을 때 쓸 수 있는 설명입니다.

1
입력 수신

사용자가 입력한 텍스트(프롬프트)를 받습니다. 이것이 AI가 처리할 전체 맥락이 됩니다.

→
2
토큰화

입력된 텍스트를 토큰이라는 작은 단위로 쪼갭니다. "안녕하세요"도 여러 토큰으로 분리됩니다.

→
3
벡터 변환

각 토큰을 수천 개의 숫자 배열(벡터)로 변환합니다. 이 숫자들이 의미와 문맥을 표현합니다.

→
4
다음 토큰 예측

수백억 개의 파라미터를 거쳐 "다음에 올 가장 적절한 토큰"을 확률로 계산합니다.

→
5
반복 생성

한 토큰씩 예측·생성을 반복해서 최종 답변이 만들어집니다. 스트리밍으로 보이는 이유입니다.

핵심은 AI가 "답을 알고 있는 게" 아니라 "가장 그럴듯한 다음 단어를 계속 예측"하는 것입니다. 학습 데이터가 방대하기 때문에 그 예측이 매우 정확하게 느껴질 뿐입니다.
PART 03

왜 프롬프트를 잘 써야 하나요? — 질문의 질이 답의 질을 결정합니다

AI의 성능은 고정되어 있습니다. 같은 AI라도 프롬프트 방식에 따라 결과 품질이 완전히 달라집니다.

나쁜 프롬프트의 패턴
  • "웹사이트 만들어줘" → 어떤 웹사이트인지 AI가 추측해야 함
  • "코드 고쳐줘" → 어디가 문제인지, 어떻게 고쳐야 하는지 불명확
  • "예쁘게 만들어줘" → "예쁜"의 기준이 사람마다 다름
  • "더 좋게 해줘" → 무엇이 좋은 것인지 AI가 판단해야 함
  • 역할 부여 없음 → AI가 적절한 수준을 모름
  • 배경 설명 없음 → 지금까지의 맥락을 AI가 모름
좋은 프롬프트의 요소
  • 역할 부여 — "너는 10년 경력 시니어 프론트엔드 개발자야"
  • 배경 설명 — "지금 카페 소개 랜딩 페이지를 만들고 있어"
  • 구체적 요청 — "헤더 + 카드 3개 + CTA 버튼이 필요해"
  • 스타일 지정 — "색상은 따뜻한 브라운 계열, 폰트는 깔끔하게"
  • 출력 형식 — "HTML/CSS 파일 분리해서 작성해줘"
  • 제약 조건 — "jQuery 쓰지 말고 순수 JS로만 해줘"
프롬프트 실력 수준
초급
막연하게 요청 "로그인 기능 만들어줘" — AI가 기술 스택, 디자인, 요구사항을 모두 추측해서 만듦
중급
조건을 붙여서 요청 "React + Tailwind로 이메일/비밀번호 로그인 폼을 만들어줘. 유효성 검사 포함해줘"
고급
맥락·역할·제약·출력형식 모두 포함 "너는 시니어 풀스택 개발자야. 지금 Next.js 프로젝트의 로그인 페이지야. 이미 만든 Button 컴포넌트를 재사용하고, 에러 상태도 처리해줘. TypeScript로 작성해줘."
PART 04

AI의 한계 — 맹목적으로 믿으면 안 됩니다

AI를 잘 활용하려면 AI가 잘하는 것과 못하는 것을 정확히 알아야 합니다. 수강생에게 반드시 전달해야 할 내용입니다.

주의 1
할루시네이션 (Hallucination)

AI가 틀린 정보를 자신있게 말하는 현상입니다. 존재하지 않는 라이브러리, 없는 함수, 잘못된 역사 사실 등을 마치 사실인 것처럼 얘기합니다.

예: "React에서 useAutoLayout() 훅을 쓰면 돼" → 실제로 없는 훅을 자신있게 알려줌
대처법: 중요한 정보는 반드시 공식 문서로 검증. 코드는 실행해서 확인.
주의 2
학습 데이터 기준 날짜

AI는 특정 시점까지의 데이터로만 학습했습니다. 그 이후에 나온 새로운 라이브러리, 프레임워크 업데이트, 최신 뉴스는 모릅니다.

예: 최근에 출시된 Next.js 새 기능을 물어보면 예전 방식으로 알려줄 수 있음
대처법: "이게 최신 방식이 맞아?"라고 한 번 더 확인. 공식 changelog 참조.
주의 3
수학·계산 오류

언어 모델이기 때문에 복잡한 수학 계산이나 정확한 숫자 연산에서 실수를 할 수 있습니다. 단순 사칙연산은 괜찮지만 복잡한 계산은 주의해야 합니다.

예: 긴 숫자 곱셈이나 통계 계산을 시키면 가끔 틀린 값을 내놓음
대처법: 중요한 계산 결과는 직접 검산하거나 계산기/코드로 검증.
주의 4
맥락 한계 (Context 초과)

대화가 매우 길어지면 AI가 앞의 내용을 잊어버리거나 일관성을 잃을 수 있습니다. 컨텍스트 윈도우를 초과하기 때문입니다.

예: 긴 코딩 세션에서 "아까 내가 말한 디자인 시스템 색상으로 해줘" → AI가 기억 못 할 수 있음
대처법: 중요한 정보는 새 대화 시작 시 다시 붙여넣기. 세션을 짧게 관리.
주의 5
개인정보·저작권

AI에 민감한 개인정보, 회사 기밀 데이터, 저작권 있는 코드를 그대로 붙여넣는 것은 주의해야 합니다.

예: 고객 데이터 파일을 AI에 업로드해서 분석 요청 → 개인정보 유출 위험
대처법: 민감 정보는 익명화 처리 후 사용. 회사 정책 확인 필수.
주의 6
AI 의존 과다

AI가 만들어준 코드를 이해하지 못한 채 그냥 쓰면 나중에 수정이나 디버깅이 불가능해집니다. AI는 도구이지 전부가 아닙니다.

예: AI가 만든 코드가 에러 날 때 "왜 에러가 나는지" 전혀 모르는 상황
대처법: AI 코드를 보면서 반드시 이해하려는 노력. 이해 안 되면 Claude에게 설명 요청.
PART 05

AI 모델 종류 — Claude · ChatGPT · Gemini, 무엇이 다른가요?

수강생이 "ChatGPT도 있는데 왜 Claude를 써요?"라고 물을 때 명확하게 설명할 수 있어야 합니다.

ChatGPT (GPT-4o)
OpenAI
  • 가장 유명하고 많이 알려진 AI
  • 이미지 생성(DALL-E), 웹 검색 기능 포함
  • 플러그인/GPTs 생태계가 풍부
  • 코딩 능력: 좋음 (단, 긴 코드에서 약점)
  • 한국어 지원: 좋음
  • IDE 연동: 별도 플러그인 필요
일반 사용에 좋음
Claude (Sonnet / Opus)
Anthropic
  • 코딩 특화 능력이 GPT 대비 뛰어남
  • 긴 코드도 끊김 없이 정확하게 작성
  • 맥락 기억(200K 토큰)이 매우 긺
  • Claude Code로 VSCode 직접 연동 가능
  • 대화하며 수정하는 방식에 최적화
  • 이유·배경까지 자세하게 설명해줌
웹 개발 학습에 최적
Gemini (Ultra / Pro)
Google
  • Google 서비스(Docs, Sheets, Gmail)와 통합
  • 최신 정보 검색과 AI 결합이 강점
  • 멀티모달(이미지·영상) 처리 능력 강함
  • 코딩 능력: 중간~좋음
  • 한국어 지원: 좋음
  • Google Workspace 사용자에게 유리
Google 생태계에 좋음
이 수업에서 Claude를 선택한 이유는 단 하나 — 웹 개발 학습과 실무에서 가장 강력하기 때문입니다. 긴 코드, 정확한 수정, 이유 설명, VSCode 직접 연동. 이 4가지가 다른 AI에 비해 월등합니다.

프롬프트를 잘 쓰는 법 — Claude에게 원하는 것을 정확히 전달하는 기술

같은 AI라도 질문 방식에 따라 결과 품질이 10배 이상 차이납니다. 프롬프트 작성법은 이 수업의 핵심 역량입니다. 멘토가 먼저 완벽히 익혀야 합니다.

PART A

좋은 프롬프트의 핵심 4원칙 — 이 4가지만 지켜도 결과가 달라집니다

역할 → 맥락 → 구체적 요청 → 반복 수정. 이 순서대로 작성하는 것이 프롬프트 기본기입니다.

01
역할 부여 (Role)

Claude에게 먼저 역할을 부여합니다. 역할에 맞는 전문 수준의 답변이 나옵니다.

예시 "너는 10년 경력 시니어 프론트엔드 개발자야"
"너는 UX 디자이너야. 사용자 친화적으로 설명해줘"
"너는 웹 개발 선생님이야. 초보자가 이해할 수 있게 설명해줘"
역할이 없으면 AI가 적절한 수준을 모릅니다. 초보자에게 고급 코드를 주거나, 전문가에게 너무 쉬운 설명을 하게 됩니다.
02
맥락 제공 (Context)

지금 무엇을 만들고 있는지, 어디까지 진행됐는지 배경을 충분히 설명합니다.

예시 "지금 카페 소개 랜딩 페이지를 만들고 있어. 헤더는 이미 만들었고 지금은 메뉴 섹션을 만들려고 해"
"이 코드는 React로 작성됐고, Tailwind CSS를 쓰고 있어. 기존 스타일을 유지해줘"
맥락이 없으면 AI가 처음부터 다시 추측해야 합니다. 많을수록 정확도가 올라갑니다.
03
구체적 요청 (Specific)

색상, 크기, 레이아웃, 포함될 요소, 기술 스택, 제약 조건을 명확하게 명시합니다.

예시 "카드 3개를 가로로 배치해줘. 각 카드에는 이미지, 제목, 설명, 버튼이 들어가야 해. 색상은 파란 계열로"
"jQuery 없이 순수 JavaScript로만 작성해줘. 외부 라이브러리 사용 금지"
막연한 요청은 막연한 결과를 낳습니다. "예쁘게 만들어줘"는 기준이 없습니다.
04
반복 수정 (Iterate)

첫 결과가 완벽하길 기대하지 않습니다. 대화하며 계속 수정해나가는 것이 올바른 방식입니다.

예시 "버튼 색을 파란색으로 바꿔줘"
"카드 간격이 너무 좁아. 더 넓게 해줘"
"모바일에서 1열로 쌓이도록 반응형 추가해줘"
대화형 AI의 진짜 강점은 반복 수정입니다. 포기하지 말고 계속 요청하세요.
PART B

나쁜 프롬프트 vs 좋은 프롬프트 — 실제 예시로 배우는 차이

같은 목적이라도 어떻게 요청하느냐에 따라 결과가 완전히 달라집니다. 수강생에게 이 예시들을 직접 보여주세요.

나쁨 "웹사이트 만들어줘" 어떤 웹사이트인지, 어떤 기술을 써야 하는지 AI가 전부 추측해야 함
좋음 "너는 시니어 프론트엔드 개발자야. 카페 소개 랜딩 페이지를 HTML/CSS로 만들어줘. 헤더에는 카페 이름과 네비, 메인에는 배경 이미지와 문구, 아래에는 메뉴 카드 3개가 있어야 해. 색상은 따뜻한 브라운 계열" 원하는 결과에 가까운 코드가 바로 나옴
나쁨 "코드 고쳐줘" 어디가 문제인지, 어떻게 고쳐야 하는지 AI가 모름
좋음 "이 코드에서 버튼 클릭 시 모달이 열려야 하는데 안 열려. 에러 메시지는 'Cannot read properties of null'. 아래 코드에서 문제를 찾아서 수정해줘: [코드 붙여넣기]" 정확한 원인 파악 후 수정 코드 제공
나쁨 "더 예쁘게 만들어줘" "예쁜"의 기준이 없어서 AI가 임의로 판단함
좋음 "카드에 hover 시 살짝 위로 올라오는 효과 추가해줘. 그림자도 더 진하게. 색상은 지금 파란색 계열 유지하면서 좀 더 밝게 해줘" 구체적인 방향이 있어서 의도에 맞는 결과 나옴
나쁨 "React로 쇼핑몰 만들어줘" 범위가 너무 넓어서 AI가 기본 구조만 만들거나 과도하게 복잡하게 만듦
좋음 "React로 상품 목록 페이지만 만들어줘. 상품 카드는 이미지, 이름, 가격, '장바구니 담기' 버튼으로 구성. 카드 클릭 시 상세 페이지로 이동. useState로 장바구니 개수 표시. Tailwind 사용" 명확한 범위와 요구사항으로 바로 사용 가능한 코드 제공

나 (수강생)

STEP 2 — 피드백 & 수정
나 (수강생)
"좋아. 근데 카드 간격이 너무 좁고, 버튼 색이 너무 어두워. 카드 간격 더 넓게 하고 버튼은 밝은 카멜 색으로 바꿔줘."
↓
Claude
카드 gap 수정 + 버튼 색상만 변경한 업데이트 코드 제공. 변경된 부분 주석으로 표시.
STEP 3 — 기능 추가
나 (수강생)

3개월 수료 시 도전 가능

AI 활용 프론트엔드 개발자

AI 없이 개발하는 사람과 AI로 개발하는 사람의 속도 차이는 이미 3~5배입니다. 같은 시간에 더 많이 만들고, 혼자서도 완성도 높은 서비스를 만드는 개발자 — 채용 시장에서 가장 주목받는 포지션입니다.

채용 공고에 "AI 활용 능력 우대" 명시가 빠르게 늘고 있습니다.
→ 같은 실력이라면 AI 잘 쓰는 개발자가 무조건 뽑힙니다

실제로 하는 일

  • Figma 디자인 → React 컴포넌트로 구현 (Claude로 속도 3배)
  • 반응형 웹 페이지 제작 — 모바일·태블릿·데스크톱 모두 대응
  • REST API 연동으로 실시간 데이터를 화면에 표시
  • Claude로 코드 초안 생성 → 직접 리뷰·수정·최적화
  • 에러 디버깅 — Claude에게 에러 붙여넣고 원인 파악
  • Vercel로 배포 → 실제 URL로 접속 가능한 서비스 완성

취업 가능한 포지션

프론트엔드 개발자 웹 퍼블리셔 UI 개발자 풀스택 개발자(주니어) AI 활용 웹 개발자

취업 가능한 회사

  • 스타트업 — 빠른 실행이 필요한 곳, AI 개발자 수요 급증
  • 중소 IT 기업 — React 기반 웹 서비스 유지보수 및 신규 개발
  • 웹에이전시 — 월 단위로 다양한 클라이언트 사이트 제작
  • 프리랜서 — 랜딩 페이지·쇼핑몰·포트폴리오 사이트 수주
  • 인하우스 — 기업 내부 웹 서비스 개발 담당

  • React 앱 1~2개 — Vercel 배포, 실제 접속 가능한 URL 필수
  • 반응형 랜딩 페이지 1~2개 — 디자인 완성도 높게
  • GitHub 저장소 — 커밋 내역이 쌓여있는 것 (꾸준히 만든 증거)
  • README — 기술 스택, 구현 기능, AI 활용 방식 명시

취업에 성공하는 사람의 3가지 공통점

01

포트폴리오 퀄리티로 승부한다

출신 학교·자격증·수료 기관보다 "실제로 뭘 만들었는지"가 결정적입니다. 완성도 높은 결과물 1개가 엉성한 결과물 10개보다 낫습니다. 수업에서 만드는 모든 것을 포트폴리오 퀄리티로 끌어올리세요.

수업에서 만든 것을 그냥 두지 말고, 계속 개선해서 "내 작품"으로 만드세요.
02

수료 후에도 혼자 계속 만든다

4개월 수업은 시작점입니다. 수업이 끝난 후에도 매주 무언가를 만드는 습관이 있는 사람이 취업에 성공합니다. GitHub 커밋 내역이 꾸준히 쌓이는 것 자체가 실력의 증거입니다.

수료 후 2개월 안에 취업에 성공한 사람들의 공통점 — 수업 끝나도 매일 코딩했습니다.
03
