### 2. AI + Claude로 웹을 해야 하는 이유

**AI를 알아야 하는 이유**
- 단순 반복 업무(회계, 디자인, 번역, 문서)는 이미 AI로 대체 중
- 코드만 짜는 개발자는 AI로 대체 가능 — AI를 다루는 개발자만 살아남음
- AI 쓰는 사람 1명 = AI 안 쓰는 사람 3~5명의 생산성
- AI는 선택이 아닌 생존 도구 — 지금 당장 실무에서 쓰이고 있음
- **결론**: AI를 모르면 어떤 직군이든 도태, AI를 알면 어떤 직군이든 경쟁력

**Claude가 좋은 이유**
- 코딩 능력이 GPT 대비 뛰어남 (긴 코드, 정확한 수정)
- 맥락 기억력이 길어서 대화하며 만들기 좋음
- Claude Code로 VS Code와 직접 연동 가능

**AI가 어떻게 학습하는가**
- LLM(거대 언어 모델) 개념 간단 설명
- 왜 프롬프트를 잘 써야 하는지

**프롬프트 잘 쓰는 법**
- 역할 부여 ("너는 시니어 프론트엔드 개발자야")
- 맥락 제공 (배경 설명)
- 구체적으로 요청 (막연하게 X)
- 반복 수정으로 완성도 높이기

## 월차별 세부 내용

### 1개월차 — AI 기반 웹 제작 환경 구축

**웹디자인 기본**
- 웹 구조 이해 (헤더, 본문, 푸터)
- 색상 — 주색/보조색/배경색 조합, 대비(contrast) 개념
- 폰트 — 가독성, 크기 위계 (제목/본문/캡션)
- 여백 — 요소 간 간격이 왜 중요한지
- 좋은 웹 vs 나쁜 웹 사례 비교

**Claude 세팅**
- Claude.ai Pro 플랜 가입
- VS Code 설치 및 기본 세팅
- Claude Code(CLI) 설치 + VS Code 연동
- 기본 사용법 (파일 열기, 대화 시작, 명령 입력)

**프롬프트로 웹 만들기**
- UI 컴포넌트 단위로 만들기 — 버튼, 카드, 슬라이더, 아이콘 배치
- 컴포넌트 조합해서 한 페이지 완성
- 프롬프트로 수정 요청하는 법 (색 바꾸기, 크기 조정, 위치 변경)
- 자기소개 페이지 제작 (이름, 소개글, 사진, SNS 링크)
- 랜딩 페이지 제작 (히어로 섹션, 특징 소개, CTA 버튼)

## 제작 원칙

**콘텐츠**
- 이모지 사용 금지 — HTML, CSS, JS 어디에도 이모지 넣지 말 것. 텍스트와 아이콘 폰트만 사용.

**디자인**
- 톤: 밝고 깔끔한 화이트 기반
- 주색상: 파란 계열 (`#2563EB`)
- 폰트: Pretendard 또는 Noto Sans KR
- 레이아웃: 여백 충분히, 섹션 구분 명확하게
- 반응형: 모바일 / 데스크톱 모두 대응

**코드**
- CSS 변수 사용 — 색상/폰트/간격을 `:root`에 몰아서 정의, 수정 시 한 곳만 바꾸면 됨
- 섹션 단위 분리 — 각 섹션을 독립적인 블록으로 작성, 순서 변경·추가·삭제 쉽게
- 클래스 네이밍 — 역할 기반으로 명확하게 (`.section-title`, `.card-item` 등)
- 주석 — 각 섹션 시작에 구분 주석 달기 (`/* === 섹션명 === */`)
- JS는 `script.js`, CSS는 `style.css` 별도 파일로 분리
- 월차 카드/섹션은 복붙해서 쉽게 추가할 수 있는 구조로
